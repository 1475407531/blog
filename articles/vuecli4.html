# 使用vue cli4.x（脚手架）搭建vue项目的过程详解

# Vue介绍

https://cn.vuejs.org/v2/guide/ (opens new window)

Vue是一套用于构建用户界面的渐进式JavaScript框架。与其它大型框架不同的是，Vue 被设计为可以自底向上逐层应用。Vue 的核心库只关注视图层，不仅易于上手，还便于与第三方库或既有项目整合。另一方面，当与现代化的工具链以及各种支持类库结合使用时，Vue 也完全能够为复杂的单页应用提供驱动。

# 什么是Vue Cli？

https://cli.vuejs.org/zh/guide/ (opens new window)

Vue Cli是一个基于Vue.js进行快速开发的脚手架工具，其作用就是用配置好的模板迅速搭建起一个项目工程来，省去自己配置webpack配置文件的基本内容。

# vue-cli搭建项目

一、环境准备

Node 版本要求 Vue CLI 4.x 需要 Node.js v8.9 或更高版本 (推荐 v10 以上)。你可以使用 n，nvm 或 nvm-windows 在同一台电脑中管理多个 Node 版本。

由于 Node.js v8 已不再维护，我们建议使用 Node.js v10 以上的版本，以保证最佳的兼容性。

使用命令安装 vue cli包

npm install -g @vue/cli
# or
yarn global add @vue/cli

安装成功后，可以用命令检查其版本是否正确：

vue --version
# or
vue -V

二、创建项目并选择配置

1、创建项目 找到一个存放项目的文件夹使用命令,回车创建项目：

vue creat <projectname>

2、选择配置方式

这里有几种选择，一般我们选择最后一项，根据项目需求手动配置要用到的。（使用↑ ↓箭头切换，Enter确认，）

3、选择配置

需要什么自己选择（上下箭头切换 空格选中 ）babel:转换es6语法的这个必须要，typescript暂时没涉及到，router管理路由的也选上，vuex状态管理就看自己需不需要用，css pre样式预处理器这个也加上，linter/Formatter统一代码风格，最后2个分别是单元测试和端对端测试，一般不会用到。选择项目用到的几个常用的 然后回车确认。

（1）、vue版本选择

Vue 3尚未正式发布，但是维护者已经发布了Beta版本，项目用API完善的vue 2。

（2）、路由是否使用历史模式选择

路由模式分为Hash模式 History(历史)模式 Abstract模式 三种 hash模式最明显的特点就是url上会带有#号， History历史模式的url就跟正常的访问地址一样。

如果使用历史模式，部署生产是运维服务器也要做响应修改，这里选n。

（3）、选择css预处理器

这里选择css预处理器，我选的是less

（4）、选择语法检测工具，自动化代码检测可以检测出代码中的语法问题，确保输入代码语法正确，风格一致。

  • ESLint with error prevention only: 只检测语法错误
  • ESLint + Airbnb config: 独角兽公司的Airbnb，有人评价说“这是一份最合理的JavaScript的编码规范” 几乎涵盖了js的各个方面。
  • ESLint + Standard config: standardjs是一份强大的js编码规范，自带linter和自动代码纠正，没有配置，自动格式化代码，可在编码早期发现规范问题和低级错误。
  • ESLint + Prettier: Prettier座位代码格式化工具，能够统一整个团队的代码风格。我这里选择的是ESLint + Prettier

(5)、选择语法检查方式

  • Lint on save: 保存就检测
  • Lint and fix on commit: 用户提交文件到git的时候检测，我这里选择Lint on save,代码保存时就检测。

(6)、选择配置文件放在哪里（babel,postcss eslint这些配置文件）

  • In dedicated config files: 独立配置文件，每个配置文件都单独管理，我这里选择In dedicated config files
  • In package.json: 放在package.json文件里

(7)、对于以上的配置是否保存，如果选择Y，下次搭建项目时候就会出现这一次的配置选项，如果选n，下次构建项目重新像这次重新从头配置。

这里选择n，每次项目可能用到的不一样，就每次重新选择配置。
